# The Router

**AGG (Aggregator)** is the "Router" for the 402 (Payment Required) economy. It abstracts away the complexity of blockchain payments, allowing users to pay with any token on any chain.

## Routing Architecture

AGG functions like a DEX aggregator (Jupiter) but for **payments**, not just swaps. It solves the "n-to-1" problem: Users have `n` tokens on `m` chains, but the Merchant wants `1` specific token on `1` chain.

### 1. The 402 Handshake
The protocol begins when a client hits a gated resource.

**Server Response (402 Payment Required)**
```http
HTTP/1.1 402 Payment Required
WWW-Authenticate: 555x402 realm="sector-13",
  amount="5.00",
  token="USDC",
  chain="solana",
  recipient="HuW...9zP"
```

### 2. Pathfinding (The Dijkstra Algorithm)
The AGG Client (SDK) calculates the optimal route based on the user's wallet state.

**Factors:**
*   **Gas Cost**: Solana (<$0.001) vs Base ($0.05) vs Polygon ($0.02).
*   **Swap Slippage**: If paying with SOL to settle in USDC.
*   **Bridge Time**: If paying from Base to settle on Solana (via CCTP).

**Example Route:**
`User (Base/ETH) -> Uniswap (USDC) -> CCTP Bridge -> Solana (USDC) -> Merchant`

### 3. Solana Settlement (v0 + LUTs)
On Solana, AGG uses **Versioned Transactions (v0)** and **Address Lookup Tables (LUTs)** to pack complex logic into a single atomic transaction.

**Transaction Structure:**
1.  **Swap Instruction**: Jupiter CPI to convert User Token -> USDC.
2.  **Fee Instruction**: Transfer 1% Platform Fee to Treasury.
3.  **Payment Instruction**: Transfer 99% to Merchant.
4.  **Memo Instruction**: Attach `orderId` for reconciliation.

```rust
// Anchor Instruction: Settle Payment
pub fn settle_payment(ctx: Context<Settle>, amount: u64) -> Result<()> {
    // Atomic Swap & Transfer
    token::transfer(ctx.accounts.user, ctx.accounts.merchant, amount)?;
    emit!(PaymentSettled { id: ctx.accounts.order.id });
}
```

## Fee Structure

AGG enforces a transparent fee model to sustain the ecosystem.

| Fee Type | Rate | Payer | Description |
| :--- | :--- | :--- | :--- |
| **Platform Fee** | 1.0% | Merchant | Deducted from the settlement amount. |
| **Gas Fee** | Variable | User | Network transaction costs (abstracted via Relayer if gasless). |
| **Facilitator Fee** | 0.1% | User | Paid to the node that constructs the transaction (if using a Relayer). |

## Gasless & Cross-Chain Settlement

AGG abstracts the complexity of blockchain interactions, ensuring users never need to worry about holding native gas tokens (ETH, SOL, MATIC) or bridging assets manually.

![Gasless Flow](/images/gasless-flow.jpeg)

### Gasless Transactions (Meta-Transactions)
To enable a true "Web2-like" experience, AGG utilizes a network of Relayers.

1.  **User Intent**: The user signs a typed data message (EIP-712 or Solana equivalent) authorizing a payment. This signature **does not** require gas.
2.  **Relayer Dispatch**: The signed message is sent to an AGG Relayer.
3.  **Execution**: The Relayer submits the transaction to the blockchain, paying the gas fee.
4.  **Reimbursement**: The smart contract deducts the gas cost + a small fee from the user's payment token (e.g., USDC) and reimburses the Relayer.

### Cross-Chain Settlement (CCTP)
For payments originating on other chains (Base, Polygon, Ethereum), AGG leverages Circle's CCTP for capital-efficient bridging.

1.  **Burn (Source)**: User burns USDC on the source chain (e.g., Base).
2.  **Attest (Circle)**: Circle's attestation service verifies the burn event.
3.  **Mint (Destination)**: The AGG Relayer submits the attestation to Solana, minting the equivalent USDC directly into the settlement flow.
4.  **Finalize**: The minted USDC is routed to the Merchant (or converted to $555).

*Latency: ~15 seconds (Finality dependent).*
