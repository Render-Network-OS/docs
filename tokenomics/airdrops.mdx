<AccordionGroup>

## Claiming Process

### Multi-Phase Distribution

<Info>
  **Phase 1:** Early Adopter Claims (Days 1-30) **Phase 2:** Content Creator
  Claims (Days 31-60) **Phase 3:** Community Engagement Claims (Days 61-90)
  **Phase 4:** Staking & LP Claims (Days 91-120) **Claim Portal:**
  https://claim.rendernet.work
</Info>

### Claiming Requirements

<AccordionGroup>
  <Accordion title="Wallet Setup" icon="wallet">
    **Technical Requirements:**

    **Supported Wallets:**
    - **Phantom:** Recommended for seamless claiming
    - **Solflare:** Full-featured wallet support
    - **Trust Wallet:** Mobile wallet compatibility
    - **Ledger/Trezor:** Hardware wallet support

    **Wallet Preparation:**
    - **SOL Balance:** Minimum 0.01 SOL for transaction fees
    - **RNDR Storage:** Ensure wallet can receive SPL tokens
    - **Network:** Connected to Solana Mainnet
    - **Backup:** Secure wallet backup and recovery phrase

  </Accordion>

  <Accordion title="Eligibility Verification" icon="shield-check">
    **Identity & Eligibility Verification:**

    **Verification Steps:**
    1. **Wallet Connection:** Link wallet to claim portal
    2. **Category Selection:** Choose eligible airdrop categories
    3. **Proof Submission:** Provide required proof for each category
    4. **Identity Verification:** KYC for allocations over 50,000 RNDR
    5. **Final Confirmation:** Review and confirm claim amounts

    **Required Documents:**
    - **Social Proof:** Screenshots of social media accounts
    - **Activity Proof:** Platform usage statistics and history
    - **Wallet History:** Transaction records for staking/LP participation
    - **Identity Documents:** Government ID for KYC verification

  </Accordion>

  <Accordion title="Claim Transaction" icon="send">
    **On-Chain Claiming Process:**

    **Transaction Details:**
    - **Network Fee:** ~0.0001 SOL per claim transaction
    - **Token Transfer:** Direct RNDR transfer to claiming wallet
    - **Confirmation Time:** 5-15 seconds on Solana
    - **Batch Claims:** Multiple categories in single transaction

    **Security Measures:**
    - **Multi-signature:** Claims require multiple confirmations
    - **Rate Limiting:** Prevents spam and abuse
    - **Audit Trail:** All claims logged on-chain
    - **Emergency Pause:** Ability to pause claims if issues detected

  </Accordion>
</AccordionGroup>
