# Roadmap

The Render Network Protocol roadmap is organized by **dependency order**, not calendar dates. Each phase expands the system across distribution, proof, settlement, and participation. No phase ships until its dependencies are complete.

## Phase 0: Foundation Hardening (Current)

Solidifying the base infrastructure and shipping core products.

*   **555stream**: Browser-based streaming with everywhere-all-at-once distribution, L-Bar ads, AI-triggered placement.
*   **Arcade**: 20 games with Proof of Engagement, cross-game leaderboards, points/credits system.
*   **sw4p**: CCTP V2 bridge live on Solana/Base/Polygon. 5 independent security audits complete.
*   **Alice**: Autonomous agent on Milaidy/ElizaOS v2 with 12+ plugins.
*   **VAP v1**: Heartbeat-based engagement verification, state channels, on-chain settlement.
*   **Economic Model**: 10% ARP, 50/50 split, weekly USDC settlement, structural burns.

## Phase 1: Ownership Layer

Giving creators and users verifiable ownership of their attention and assets.

*   **P1-A**: cNFT Audience Inventory — mint verified attention as compressed NFTs via Metaplex Bubblegum.
*   **P1-B**: Lock verification — on-chain proof of token locks for staking and governance.
*   **P1-C**: Hyperlink completion — full embedded wallet support, advanced analytics, referral attribution.

**Depends on**: Phase 0 completion (stable points/credits, reliable VAP).

## Phase 2: Economic Expansion

Expanding the economic model with new instruments and deeper tokenomics.

*   **VAP v2 Deployment**: Epoch-based system (6-hour epochs), attention credits, stake-weighted participation, burn loop.
*   **Creator Tokens**: Launch via pump.fun with enforced splits (1% fee, 10% ARP, 20% buybacks).
*   **Micropayments**: Sub-cent payments for granular content access.
*   **Value Taxonomy**: Classification system for different types of verified attention.

**Depends on**: Phase 1 (cNFT infrastructure, lock verification).

## Phase 3: Platform Expansion

Scaling the ecosystem with new participants and markets.

*   **Advertiser Self-Serve**: Full self-service ad platform with campaign creation, targeting, and reporting.
*   **Prediction Market**: On-chain prediction markets for stream events and game outcomes.
*   **Marketplace**: Trading of Audience Inventory cNFTs.
*   **Multi-Agent**: Multiple Alice instances with specialized roles (community, trading, content).

**Depends on**: Phase 2 (VAP v2, creator tokens, economic instruments).

## Phase 4: Protocol Evolution

Decentralization and broader ecosystem integration.

*   **Fiat Integration**: Direct fiat on/off ramps for USDC settlement.
*   **Decentralization**: Progressive decentralization of VAP verification nodes.
*   **RNDRS Protocol**: Full protocol specification for third-party implementations.
*   **Cross-Protocol**: Integration with other attention and content protocols.

**Depends on**: Phase 3 (mature marketplace, stable multi-agent operations).

---

<Note>
This roadmap uses zero-date dependency sequencing. There are no timeline dates — only dependency order. Each phase ships when its prerequisites are complete and validated.
</Note>
