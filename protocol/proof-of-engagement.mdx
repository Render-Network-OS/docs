# Proof of Engagement

**Proof of Engagement (PoE)** is the consensus mechanism for the Attention Economy. It ensures that rewards are only distributed for genuine activity — whether from humans or AI agents.

## The Heartbeat Mechanism (v1 — Live)

The core of PoE is the **Heartbeat**.

1.  **Session Start**: When a user enters the Arcade, a secure session is initialized via Ed25519 wallet signature.
2.  **Pulse**: The client sends a cryptographic pulse every few seconds (interval adjusts based on trust score).
3.  **Payload**: The pulse contains:
    *   **Session ID**: Unique identifier.
    *   **Action Data**: Inputs (mouse moves, key presses) proving liveness.
    *   **State Hash**: Hash of current game state preventing replay attacks.
4.  **Verification**: The **VAP** service verifies the signature and state transition.

## State Channels

To handle the high frequency of heartbeats without clogging the blockchain, we use **State Channels**.

*   **Off-Chain**: Heartbeats are verified and aggregated off-chain by the VAP Verifier.
*   **On-Chain**: Only the result (e.g., "User X earned 500 points") is settled on Solana.
*   **Dispute Window**: A period where fraudulent claims can be challenged.

## Anti-Sybil Measures (v1)

*   **Heuristics**: Analyzing input patterns — human input has high entropy (micro-jitters), bots are too perfect or too random.
*   **Challenges**: Injecting Turing Tests (e.g., "Click the Glitch", "Press UP, DOWN, A") during gameplay.
*   **Reputation**: Long-term wallet history affects Trust Score. Higher trust = longer pulse intervals.

## Proof of Engagement v2 (Validated — Not Yet Deployed)

VAP v2 introduces a fundamentally upgraded PoE system. Research is complete with 234 tests passing and all 5 attack trees validated as SECURE.

### Entity-Agnostic Verification
The v2 protocol does not check entity type. Humans and AI agents are treated identically. The protocol verifies **contribution quality**, not **contributor identity**.

*   **ContributionProof**: 296-byte proof format containing session data, stake reference, and cryptographic signature. Entity-agnostic by design.

### Epoch-Based Settlement
Instead of continuous settlement, v2 uses **6-hour epochs** (4 per day):

1.  During each epoch, participants generate verified engagement.
2.  At epoch boundary, a batch auction settles all attention credits.
3.  Advertisers bid for verified airtime in the epoch auction.
4.  ARP (10%) is distributed to contributors from auction proceeds.

### Stake-Based Anti-Sybil

v2 replaces heuristic-only anti-Sybil with a stake-based economic model:

*   **Minimum Stake**: 55,555 $555 tokens required to participate.
*   **Stake Weight**: `ln(1 + S/27,778)` capped at 5.0. Logarithmic diminishing returns make Sybil attacks economically irrational.
*   **Activation Period**: 7-day warmup after staking.
*   **Maturation**: Full weight at 90 days of continuous stake.
