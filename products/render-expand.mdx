## Overview

Render Expand is the growth accelerator for creators. Launch a Creator Token via pump.fun, set policy bounds, and connect staking, buybacks, and Treasury to your stream and CTRL Panel.

## Launch Flow

<Steps>
  <Step title="Initialize Token">
    Define metadata, symbol, and supply policy. Tokens are launched through pump.fun with standardized liquidity and fair‑launch mechanics. Choose initial liquidity approach and basic risk parameters.
  </Step>
  <Step title="Configure Policy">
    Set minimum buyback percentage (≥20%), split bounds (e.g., 20/80 ↔ 80/20), unbonding windows, and voting thresholds. Well‑calibrated split bounds signal generosity and upside, inspiring viewers to stake and become long‑term stakeholders.
  </Step>
  <Step title="Connect Modules">
    Wire the token to CTRL Panel, 555 Engine, and the Monetization Contract to activate the ARP / Buybacks / Treasury pipeline.
  </Step>
  <Step title="Go Live">
    Publish your profile to the marketplace with verifiable engagement metrics and staking TVL so advertisers can discover you.
  </Step>
</Steps>

## Dependency Chain

Render Expand's full vision requires several infrastructure components that are currently in development:

| Dependency | Status | Description |
|---|---|---|
| **Creator Token staking infrastructure** | In development | On-chain staking contracts for creator-level tokens |
| **Monetization Contract integration** | In development | Wiring creator tokens into the ARP / Buybacks / Treasury pipeline |
| **CTRL Panel token-gating** | In development | UI support for gating content and features based on creator token holdings |

<Warning>
  Until these dependencies are complete, the full Render Expand flow (native token minting with enforced splits, staking, and governance) is not available. See the current alternative below.
</Warning>

## Current Alternative

Creators can launch tokens today via **pump.fun simulcast**. 555stream integrates pump.fun as a broadcast destination, allowing creators to simulcast their stream while launching a token through pump.fun's fair-launch mechanics. This provides immediate token launch capability without waiting for the native Creator Token infrastructure.

## Feature Roadmap

<Steps>
  <Step title="Phase 1: pump.fun Simulcast (Live)">
    Creators simulcast to pump.fun via 555stream. Token launches use pump.fun's standard bonding curve and fair-launch mechanics. No native staking or enforced splits at this stage.
  </Step>
  <Step title="Phase 2: Native Token Minting with Enforced Splits">
    Creator tokens are minted natively within the protocol. Policy bounds (minimum buyback %, split ranges, unbonding windows) are enforced on-chain via the Monetization Contract.
  </Step>
  <Step title="Phase 3: Full Staking + Governance">
    Complete staking infrastructure with creator-level governance. Token holders can vote on creator policy adjustments, and staking TVL drives discovery ranking in the marketplace.
  </Step>
</Steps>
